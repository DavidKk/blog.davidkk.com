# 网页性能优化

- 使用 HTTP2.0
- 使用 CDN
- 使用 GZIP 压缩传输
- 减少 HTTP 请求
- 使用 DNS 预解析
- 减少 DNS 解析 - 域名少一点, 浏览器会缓存
- 减少 DOM 节点
- 减少 DOM 节点的操作
  - 必要时可以先缓存, 减少不必要的查找
  - 通过 `document.createDocumentFragment` 创建临时存放空间, 进行操作再一次性修改
- 避免重定向
- 尽量避免使用 iframe - 阻塞 onload, 共享连接数, 共享线程
  - 使用时最好通过异步插入, 否则会阻塞 onload 事件
- 图片无损压缩
- 图片使用精灵图 - 合并图片, 减少连接数
- 图片延迟加载
- 图片使用 webP
  - 既支持 Alpha 透明
  - 具有更优的图像数据压缩算法, 而且拥有肉眼识别无差异的图像质量
- 代码压缩
- 减少代码量, 提取共有代码模块
